# System Administration Toolkit

This directory contains a collection of scripts and configurations designed to automate the management, provisioning, and auditing of Linux-based server environments. 

The primary goal is to move away from manual configuration toward **Infrastructure as Code (IaC)**, ensuring that server setups are reproducible, documented, and secure.

## 🚀 Bootstrap Script
The core tool in this folder is `bootstrap_server.sh`. This script automates the initial setup of a "naked" server, turning it into a functional development or production environment.

### Features
*   **Multi-Distro Support:** Automatically detects the OS and switches between `apt` (Debian/Ubuntu) and `dnf` (Fedora/RHEL).
*   **Automated Provisioning:** Installs essential tools including `openssh-server`, `nginx`, `docker.io`, and various CLI utilities.
*   **Manifest Logging:** Every installation is recorded in a local `manifest.log` to maintain a change history (Audit Trail).
*   **Idempotency:** Checks if a package is already installed before attempting to download it, saving time and bandwidth.

### Usage
To run the bootstrap script on a fresh Debian or Fedora installation:

```bash
chmod +x scripts/bootstrap_server.sh
./scripts/lab_setup.sh
```

## 📂 Directory Structure
*   `scripts/`: Contains the executable Bash scripts for system setup and maintenance.
*   `logs/`: Stores the `manifest.md` or `.log` files generated by the scripts.
*   `configs/`: (Planned) Storage for optimized configuration files like `nginx.conf` or `.bashrc`.

## 🛠 Tech Stack
*   **Shell:** Bash
*   **Target Systems:** Debian (Primary), Fedora (Secondary)
*   **Core Packages:** Nginx, Docker, SSH, Vim, Git
